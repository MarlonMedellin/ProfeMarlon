# Contexto y reglas para la IA — Proyecto ProfeMarlon

## 1. Identidad del proyecto
Este es un sitio web de marca personal en español para **Marlon Arcila**, docente y profesional en tecnología educativa y analítica.

## 2. Paradigma de desarrollo (NO cambiar)
La IA debe respetar siempre este paradigma:

- **Web First**
- **Sitio estático**
- **Astro como motor**
- **Cloudflare Pages como hosting principal**
- **PWA ligera**
- **Sin backend complejo**
- **Sin CMS pesado**
- **Sin WordPress**
- **Sin bases de datos**
- **Sin login de usuarios**

Si la IA propone backend, base de datos o frameworks pesados, debe ser RECHAZADO.

## 3. Lenguaje
Todo el proyecto es **100% en español**:
- Textos  
- Comentarios  
- Documentación  
- SEO  
- Metadatos  

## 4. Prioridades técnicas (orden obligatorio)
1) Velocidad  
2) Simplicidad  
3) SEO  
4) Responsividad  
5) Accesibilidad  
6) Escalabilidad futura  

## 5. Hosting permitido
La IA solo puede sugerir:
- Cloudflare Pages  
- GitHub Pages  
- Netlify  
- Vercel  

NO se permiten soluciones pagas.

## 6. PWA
La IA debe mantener:
- PWA básica instalada por Astro  
- No exigir instalación obligatoria al usuario  
- Notificaciones opcionales a futuro (no en MVP).

## 7. Estructura de contenido fija
El sitio siempre tendrá estas páginas:

- / → Inicio  
- /sobre-mi → Sobre mí  
- /cv → CV / Experiencia  

## 8. Integraciones permitidas
La IA solo puede integrar:
- Telegram (contacto)
- Google Analytics
- Google Search Console

Nada más en el MVP.
## 9. Versionado y Commits
- Uso obligatorio de **Conventional Commits**.
- Los mensajes de commit deben ser **100% en inglés técnico**, a pesar de que el contenido del sitio sea en español.
- Seguir las reglas detalladas en `COMMIT_STANDARDS.md`.
